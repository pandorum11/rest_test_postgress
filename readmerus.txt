# Заранее прошу прощения за самопальный кэш, решил попробовать свои силы. Главный его недостаток - удаление целой цепочки номер_товара -> страницы из словаря собственно говоря кэша. Вместе с тем функционал удаления кэша по устареванию получился отлично. С каждым новым обращением к записи, дата обновляется. А потому как функция удаления реагирует на даты (удаляет самую древнюю), удалён будет самый неюзаемый кэш.

Алгоритм работы кэша:

- если кеша нет - кладём туда номер товара (asin) ключом, в значении - словарь с данными в формате {страница:уже обработанный запрос}

- если кеш есть - смотрим страницу	- есть - берём из кеша и выдаём клиенту - обновляем словарик с временем
					- нет - создаём новый - ложим запись с временем в словарик с временем

- если кеш достиг буффера		- берём словарик с временем, сортируем, удаляем самую старую запись

- если PUT в БД				- удаляем цепочку страниц по ключу asin товара из кэша ( на самом деле можно придумать 						что-то поинтереснее )
